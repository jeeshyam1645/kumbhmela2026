3. KEY FUNCTIONAL REQUIREMENTS
3.1 Multilingual System (Priority: High)
•	Mechanism: Manual translation toggle (Not auto-Google Translate).
•	Logic:
o	User clicks "Hindi" -> Site reloads/switches to Hindi version URL.
•	Content: All static text, menus, buttons, and form labels must be translatable.
3.2 Booking & Enquiry Engine (Priority: High)
•	Model: Lead Generation Only (No Payment Gateway).
•	Checkbox Requirement: Before the "Submit Enquiry" button works, add a checkbox:
•	[ ] I have read and agree to the Terms & Cancellation Policy.
•	(Note: This protects you legally. If they dispute a refund later, you can show they checked this box).
•	Form Locations:
o	Dedicated "Contact" page.
o	"Check Availability" pop-up on Camp Detail pages.
o	"Enquire for Puja" button on Service page.
•	Form Fields:
1.	Name (Text)
2.	Mobile Number (Number + Country Code selector). Validation: Must be 10-15 digits.
3.	Check-in / Check-out (Date Picker).
4.	Number of Persons (Dropdown: 1, 2, 3, 4, 5+).
5.	Camp Preference (Dropdown: Swiss Cottage / Deluxe Tent / Dormitory).
6.	Special Needs / Puja Request (Text Area).
•	Submission Logic:
o	Frontend: Validate fields -> Submit -> Redirect to Thank You Page.
o	Thank You Page: Display message: "Thanks [Name]! We will call you within 2 hours." + Embed "Camp Tour" Video.
3.3 Notification System (The "Speed to Lead")
•	Email Alert: System sends full enquiry details to admin@yourcamp.com.
•	WhatsApp Alert (Internal): Integration (via API like Wati/Interakt or PHP script) to send a structured message to the Owner’s Mobile.
•	WhatsApp Alert (Customer - Optional): Auto-reply: "Namaste! We received your request. Our Camp Manager will call you shortly."
